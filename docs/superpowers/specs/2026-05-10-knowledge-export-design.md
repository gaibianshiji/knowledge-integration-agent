# 知识成果导出系统设计

> **目标**：为医学院老师提供高质量的知识整合成果，用于评估产品质量
> **范围**：知识提取质量优化 + 结构化报告导出 + 可交互图谱导出
> **不在范围内**：对话系统、RAG、网站部署、前端 UI

---

## 1. 问题分析

当前知识图谱覆盖率不足：生理学教材 533 章只提取了 5 章（覆盖率 0.9%），其他教材类似。老师看到覆盖率会质疑质量。

根因：`extract_textbook_knowledge` 的 `max_chapters=50` 限制，且小章节（100-500 字）单独调用 LLM 效率低。

## 2. 方案设计

### 2.1 重新提取：合并小章节，提高覆盖率

**策略**：
- 将 100-500 字的小章节按相邻顺序合并为 500-800 字的批次
- 500-2000 字的章节保持不变
- 2000+ 字的章节截断到 2000 字
- 目标：覆盖全部章节，减少 LLM 调用次数

**预计数据量**：
- 7 本教材共约 3000+ 章节
- 合并后约 1000-1500 次 LLM 调用
- 预计产出：每本 200-400 个知识点，总计 1500-2500 节点

**实现**：
- 新增 `merge_small_chapters(chapters, min_size=500)` 函数
- 修改 `extract_textbook_knowledge` 使用合并后的章节列表
- 并发控制：每次最多 10 个并发请求，避免 API 限流

### 2.2 单本教材报告（7 份 Markdown）

每份报告结构：

```markdown

- 总章节：X 章
- 已提取章节：Y 章（覆盖率 Z%）
- 知识点数：N 个
- 关系数：M 条

## 知识点体系

### {根节点名称} [分类]
├── {子节点1} [分类]
│   ├── {孙节点1} [分类] — 定义...（章节名，置信度 95%）
│   └── {孙节点2} [分类] — 定义...
└── {子节点2} [分类]

## 关系网络

### 前置依赖（X 条）
| 源知识点 | 目标知识点 | 描述 |
|----------|------------|------|

### 包含关系（X 条）
| 上位概念 | 下位概念 | 描述 |
|----------|----------|------|

### 并列关系（X 条）
| 概念A | 概念B | 描述 |
|-------|-------|------|

### 应用关系（X 条）
| 知识点 | 应用场景 | 描述 |
|--------|----------|------|

## 章节覆盖分析

| 第一章 | ✅ 已提取 | 12 |
| 第二章 | ✅ 已提取 | 8 |

```

### 2.4 可交互 HTML 知识图谱（9 个文件）

**7 个单本图谱**（`output/图谱_生理学.html` 等）：
- 内嵌 D3.js，独立文件，双击打开
- 力导向图，节点按分类着色
- 点击节点显示详情（名称、定义、章节、置信度）
- 关系线按类型区分样式（实线/虚线/颜色）

**1 个整合图谱**（`output/整合图谱.html`）：
- 所有教材整合后的图谱
- 节点按教材着色（7 种颜色）
- 保留/合并/删除的节点用不同形状区分
- 冲突节点特殊标记

**1 个总览页面**（`output/总览.html`）：
- 展示 7 本教材的基本统计
- 链接到各单本图谱和整合图谱
- 嵌入整合报告的关键数据

## 4. 实施步骤

1. **合并小章节函数** — 新增 `merge_small_chapters()`
2. **重新提取 7 本教材** — 调用 LLM，保存到 `backend/data/graphs/`
3. **重新整合** — 基于新图谱重新运行整合
4. **生成单本报告** — 7 份 Markdown
5. **生成整合报告** — 1 份 Markdown
6. **生成 HTML 图谱** — 9 个独立 HTML 文件
7. **提交到 git** — 整个 output/ 目录
